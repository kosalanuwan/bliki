Writing a JS utility library

5th September 2020
1 minute read
Tweet

Remember the stuff we picked up writing a stripped-down version of jQuery? Think of this as a stripped-down version of lodash, a mix and match of a few helper methods from utility libraries, then bundled together into global namespace as _.

It was a real hail mary, but the code isn’t complicated as other huge JS libraries like angular or babel. A couple of days and a 3-4 utility libraries on GitHub later, I was able to pulled togher my own utility belt library.

Here’s the complete source code on GitHub.

Examples in use:

/**
 * Use as typical '_' utility functions
 */
_.format('Hello {0}!', username);
_.kebabCase('Some teXt to Kebab case');
_.isblank(username);

/**
 * Or, do the same with chaining and prototype option
 */
'Hello {0}!'.format(username);
'Some teXt to Kebab case'.toKebabCase();
username.isblank();

You’d be amaze how tricky it is to enable both function and prototype options. And for that, I do dig into underscore-string utility library in order to pull out what is needed !
