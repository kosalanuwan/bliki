Write your own DOM manipulation library

12th August 2020
1 minute read
Tweet

I’d prefer to write my own code for DOM manipulation. In other words, a stripped-down version of JS libraries like jQuery. But one thing! Why the montage of “$” in the JS world when querying the DOM? After a few minutes of duckduckgo, here’s what I’ve found.

Think of it as a mix and match of a few aliases of native browser API calls like .querySelector(), .querySelectorAll() et al, then bundled together into global namespace as $.

Examples in use:

/**
 * Wire a 'click' event to the <body> tag
 */
var bodyEl = $('body');
$on(bodyEl, 'click', showContentIfRouteLink);

/**
 * Do the same with chaining and prototype option
 */
$('body').on('click', showContentIfRouteLink);

You’d be amaze how difficult it is to find out the pieces. And, for that, I do dig into those opensourced libraries in order to pull out what is needed !
