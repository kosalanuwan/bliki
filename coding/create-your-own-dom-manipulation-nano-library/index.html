Create your own DOM manipulation nano-library

12th August 2020
1 minute read
Tweet

I’m a huge nerd of javascript (JS). I’d use my own nano-library for DOM manipulation instead of adding huge JS frameworks and libraries like jQuery, Lodash, Underscore.js, et al.

If you want to understand at the most basic level, here’s the complete source code I pulled together.

And, for that, I do dig into those frameworks and libraries in order to pull out what is needed, with of course. Luckily, they are all opensourced!

Example usage:

/**
 * Attach the login function to the menu item Log in.
 */
$('a[data-route-link="/login"]')
  .parent('div')
  .on('click', vm.loginWithRedirect);

/**
 * Validate the input text box if it's mandatory.
 */
$$('input[required]').on('input', checkValidity);

What You’ll Learn

Think of it as a mix and match of a collection of helper functions for DOM manipulation bundled into a single namespace. In this case, the it would be the global namespace or the window object.

  • First, I use IIFE pattern, a bit similar to Revealing Module pattern to wrap the library
  • Then, the Polyfill pattern is used for helper functions, and
  • Finally, I use Chaining as for fluent use of helper functions, and
  • I follow the JSDoc style for documenting the javascript

Run from Source

For minimum requirements, I use following tools.

  • An editor, like VS Code
  • A development local server, like Live Server for VS Code, and
  • A web browser, like Brave as for my javascript playground

You’ll want to copy the source and paste it on the browser console, then play around with those methods by calling it.
