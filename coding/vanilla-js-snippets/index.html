I’m a huge nerd of vanilla javascript (JS). I’ve built a few website on it, as well as a few libraries and still have a long way to go.

Have a look at this repository which servers as a library of boilerplate code for all things vanilla JS.

Provide Feedback

If you were wondering, ask questions and problems on GitHub. It is as easy as creating a new issue. If not, here are the ways you can provide feedback.

  • Submit bugs and request code snippets
  • Upvote popular code snippets
  • Review source code and documentation

Run from Source

I use VS Code and Brave as my javascript playground. First, you need followings to get started.

  • VS Code with Live Server extension, or
  • Web browser with Developer Tools

If you want to understand how the snippets work at the most basic level, you’ll want to copy the source, paste it on the browser console, and run the examples.

Utility Libraries

You want to try out a few major utility libraries for starters.

  • Lodash
  • Underscore, or
  • Underscore.string

But, I don’t enjoy adding them just for the sake of developer ease. Nowadays, vanilla javascript already cover most of it anyways and polyfills are there whenever some random browser doesn’t have the support.

Preferred approach

I prefer mix and match snippets, helper methods, boilerplate code, polyfills et. al..

For that, I do dig deep into any javascript code, libraries, frameworks in order to pull out what is needed, with of course. Luckily, they are all opensourced!

  • I peek into Financial Time’s polyfill library
  • The Vanilla JS Toolkit site is a good source of information too
