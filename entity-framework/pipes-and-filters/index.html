Writing pipes and filters to query data

5th November 2009
1 minute read
Tweet

LINQ may look like the best thing every happend to IEnumerable types but it’s getting all kinds of help from our friends at C#. For the record, it’s the Extension Methods that enabled query functionality without creating new derived types or modifying original types of enumerables.

I like the idea of extending predefined types with extension methods for domain-specific functionality. It gives the whole thing a very Ubiquitous feel since the LINQ queries are more readable, say, two months from now like Uncle Bob say on Clean Code.

Example 1

Use of pipes and filters to get a list of trending products belong to the supplier.

db.Products
  .ForSupplier(supplierId)
  .Trending(size)
  .ToList();

The first thing we are going to need is reusable Lambda expressions. I can consolidate user-defined expressions into methods so a little extract method magic is all that is necessary to get started. Kids’ stuff.

Example 2

User-defined reusable query expressions.

namespace Application.Products.Queries
{
    public static class QueryExtensions
    {
        public static IQueryable<Product> ForSupplier(
            this IQueryable<Product> query, int id) =>
            query.Where(p => !p.Discontinued && p.SupplierId = id);

        public static IQueryable<Product> Trending(
            this IQueryable<Product> query, int size) =>
            query.OrderBy(p => p.ModifiedDate)
                 .Take(size);
    }
}

Next is chaining IQueryable types. It’s slightly obnoxious that extension methods require non-nested non-generic static classes. I’m gonna go ahead and say there’s no use of instance methods syntax so there’s no way of injecting dependencies. I have to break out same extract methods into static methods and pass in the IQueryable domain-specific entity type as the first parameter which the method must operate on. All I have to do is add the this modifier to the parameter and the C# compiler will do the rest for me. I’m not going to lie, you’d be amazed how tricky it is and if you return the wrong type, chaining breaks and nothing returns.
